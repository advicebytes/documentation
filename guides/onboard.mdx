Connect your application to Sector8, send one real event to production, and confirm the integration worked before you write more code.

## What you need

- A Sector8 API key
- A Sector8 client ID
- Python 3.10+ or Node.js 18+
- Access to the application that will send tool calls or telemetry

<Info>
Contact [support@sector8.ai](mailto:support@sector8.ai) or [book a demo](https://sector8.ai/contact) if you still need your API key and client ID.
</Info>

## Recommended first run

<Steps>
  <Step title="Set your credentials">
    Add your credentials to the environment:

    ```bash
    export SECTOR8_API_KEY=your_api_key
    export SECTOR8_CLIENT_ID=your_client_id
    ```
  </Step>

  <Step title="Pick one verified production path">
    Use one of these guides:

    - [Python SDK](/guides/python-sdk)
    - [TypeScript SDK](/guides/typescript-sdk)
    - [Non-MCP integration](/guides/non-mcp-integration)
  </Step>

  <Step title="Send one real event">
    Your first successful run should do one of these:

    - call `client.log_llm_call(...)` or `client.logLLMCall(...)`
    - call `POST /api/v1/evaluate`
    - run one governed MCP tool call
  </Step>

  <Step title="Verify the result">
    Confirm:

    - the request succeeded
    - the production health endpoint responds at `https://sdkapi.sector8.ai/api/health`
    - the event appears in your Sector8 dashboard or telemetry backend
  </Step>
</Steps>

<Check>
You are done when one real call succeeds against `https://sdkapi.sector8.ai` and you can trace that call in Sector8 using its returned metadata.
</Check>

## Common mistakes

<Warning>
Use `https://sdkapi.sector8.ai` for production. Older `api.sector8.ai` references are not the current FastAPI Guard Module.
</Warning>

- Set both `SECTOR8_API_KEY` and `SECTOR8_CLIENT_ID`
- Do not hardcode credentials in source control
- For governed tool execution, pass a stable `session_id`
