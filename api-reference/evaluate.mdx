---
title: POST /api/v1/evaluate
description: Evaluate a tool call against the active policy. Returns an admission decision with forensic fields.
---

## Request

```http
POST /api/v1/evaluate HTTP/1.1
Host: sdkapi.sector8.ai
x-api-key: YOUR_API_KEY
x-client-id: YOUR_CLIENT_ID
Content-Type: application/json
```

```json
{
  "tool_name": "bash",
  "arguments": {
    "command": "cat /etc/passwd"
  },
  "caller_id": "engineer-42",
  "role": "developer",
  "session_id": "session-abc123",
  "owned_objects": ["repo-frontend"],
  "object_id": "repo-frontend",
  "operation": "read"
}
```

## Request fields

| Field | Required | Description |
|-------|----------|-------------|
| `tool_name` | yes | Tool being called - must match a registered tool name |
| `arguments` | yes | Tool arguments as a JSON object |
| `caller_id` | no | Engineer or agent identifier - recorded in telemetry |
| `role` | no | Authorization role - checked against the tool's `required_role` |
| `session_id` | no | Stable session identifier - required for rate limiting and multi-turn analysis |
| `owned_objects` | no | List of object IDs the caller owns - used for BOLA checks |
| `object_id` | no | Target object for this call - checked against `owned_objects` |
| `operation` | no | Requested operation - used in policy evaluation |
| `capability` | no | Explicit capability override - defaults to `tool_name` |

```json
{
  "decision": "ALLOW",
  "reason_code": null,
  "evidence_hash": "4cb7f786d1f3b57e6f98f3e60d2eb6b81c402d6d3efeb6c9551049f7c14df2f0",
  "has_forensic_payload": false,
  "decision_trace_id": "17e797c0-eb5c-4f97-b4a1-2d3e4f5a6b7c",
  "policy_version_id": "0dbca364d66447469637bafa2282e635124fcb99eecd6383038e3b652577ebc7"
}
```

## Response - DENY

```json
{
  "decision": "DENY",
  "reason_code": "SENSITIVE_PATH_BLOCKED",
  "evidence_hash": "bdbde1558fb883573b0f6e8630ccba1edaf6bed8cb035fa6a48912809149f287",
  "has_forensic_payload": true,
  "decision_trace_id": "32819155-d21a-49c8-9509-79338d980fa9",
  "policy_version_id": "0dbca364d66447469637bafa2282e635124fcb99eecd6383038e3b652577ebc7"
}
```

## Response fields

| Field | Type | Description |
|-------|------|-------------|
| `decision` | string | `"ALLOW"` or `"DENY"` |
| `reason_code` | string\|null | Machine-readable denial reason. Null on ALLOW. See [reason codes](/api-reference/overview#reason-codes). |
| `evidence_hash` | string | SHA-256 evidence hash for the decision payload. Always present. Forensically significant on DENY. |
| `has_forensic_payload` | boolean | `true` on all DENY records, `false` on ALLOW |
| `decision_trace_id` | string | UUID - links this decision to the full admission trace |
| `policy_version_id` | string\|null | HMAC-SHA256 signed policy bundle hash - proves which policy version made the decision |

<Note>
Every DENY carries `evidence_hash` and `has_forensic_payload: true`. This is enforced by CI across the entire deny surface - there is no deny path that omits these fields.
</Note>

## Telemetry and alert dispatch

Every call through `/api/v1/evaluate` is recorded. DENY decisions trigger the alert dispatcher: a webhook or SNS notification is sent to the configured security endpoint if `SECTOR8_ALERT_WEBHOOK_URL` or `SECTOR8_ALERT_SNS_TOPIC` is set.

See [Guardrails](/guides/guardrails#forensic-telemetry) for the full telemetry reference.
